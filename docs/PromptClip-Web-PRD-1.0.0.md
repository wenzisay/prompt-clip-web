# 个人 Prompts 管理Web应用产品需求文档 (PRD)


## 一、项目概述

### 1.1 产品定位

一款面向 AI 重度用户的个人 Prompt 管理工具，帮助用户高效保存、组织、复用和优化 AI 提示词，提升跨平台 AI 工具的使用效率。

更远的发展，可以做成AI技能库，不限于：prompts、skills、rules等。以及prompts精选、分享社区，甚至商店。

### 1.2 目标用户

- **核心用户**：每天频繁使用 AI 工具（ChatGPT、Claude、Midjourney 等）的专业人士
- **次级用户**：内容创作者、开发者、营销人员、产品经理、设计师、金融研究员等。
- **潜在用户**：AI 学习者和爱好者、普通用户。

## 二、 核心价值主张

用户的根本需求是：**在需要时快速找到并使用正确的 Prompt**。因此产品应聚焦于：

- **快速访问**：比打开笔记 app 更快
- **智能组织**：比文件夹更聪明
- **无缝集成**：与 AI 工具的工作流无缝衔接
- **多端同步**：第一版先实现 Web 版本，。后续多端适配与同步：APP、MAC/Win、Web、小程序、浏览器插件。
- 本地优先：prompt 存储在本地，md 格式。


## 三、产品设计原则


**快速至上**：从打开 app 到使用 prompt，不超过 3 秒
**极简主义**：界面简洁，功能直观，零学习成本
**隐私优先**：本地优先，云同步可选（第一版不做同步）


## 四、产品功能规划

### 4.1 MVP 核心功能（V1.0）

#### 4.1.1 首页

- 左右布局，左边是侧边栏，右边的 Prompt 列表。
- Prompt按照卡片/瀑布流形式组织，减轻记录的摩擦。滚动加载更多卡片。
- 侧边栏包括：导航（全部、最近使用、收藏），置顶标签、标签。
- 标签支持筛选。

#### 4.1.2 新建/编辑Prompt

- **创建与编辑**：纯文本编辑器，支持 Markdown预览。
	- 包括：标题、正文、标签
- **字符计数**：实时显示长度
- 支持历史版本保留与查看。

#### 4.1.3 快速查找

- **快速切换**：按照标题快速切换（类似obsidian中的切换笔记）
- **搜索**：全文搜索 + 标签筛选

#### 4.1.4 快速使用

- **快速复制**
- prompt 展示复制的次数。
- 支持收藏 prompt

#### 4.1.5 标签
- 支持多级标签（最多 10 级），例如：`#计算机/Linux`。按树状结构展示。
- 标签管理：删除标签（包括仅删除标签不删除 prompt、删除标签同时删除对应的 prompt）、编辑名称与图标、置顶标签。
- 
#### 4.1.6 管理
- **批量操作**：批量编辑、导出、删除



#### 4.1.5 数据管理

- 设计理念：本地优先、文件优先于应用。
- **prompt数据存储**：本地 markdown 文件存储，标签存储在 md 的元数据区，格式与 obsidian 兼容。默认存储到`~/promptclip/data`目录下，也可以手动置顶数据存储目录。这样做有个好处，即使没有应用，也可以正常查看prompt。甚至可以把 obsidian 的某个文件夹指定为数据目录，这样可以让 obsidian 无缝集成。
- **索引数据**： prompt与索引分开存储。
- **导出/导入**：JSON  / Markdown / CSV




#### 4.1.6 关键交互设计

- 首页设计
	- **顶部**：左侧搜索框（始终可见）、右侧新建按钮。
	- **中部**：全部、最近、收藏（快速访问）
	- **底部**：Prompt卡片列表
- Prompt 详情页
	- **顶部**：标题（可编辑）
	- **中部**：Prompt 内容（可编辑）
	- **底部**：标签（可编辑）


## 六、技术架构

### 6.1 UI 设计规范

UI 设计参考 `docs/promptclip-web.html` 原型，采用现代极简风格。

#### 设计系统

| 类别 | 规范 |
|------|------|
| **色彩系统** | 使用 CSS 变量定义主题色，支持亮色/暗色模式 |
| **主色调** | 蓝色 #0058bc (accent)、紫色 #4c4aca (secondary) |
| **字体** | SF Pro / 系统字体，代码使用 SF Mono |
| **图标** | Material Symbols Outlined |
| **圆角** | 卡片 14px，按钮 6-8px |
| **阴影** | 卡片使用微妙阴影，hover 时加强 |

#### 核心组件

- **侧边栏** (260px)：Logo + 导航 + 标签树
- **卡片网格**：自适应列布局，最小 300px
- **侧滑详情面板** (480px)：从右侧滑入
- **命令面板**：Cmd+K 快捷调用
- **标签颜色**：蓝/紫/紫罗兰/灰 四色分类

### 6.2 技术选型

| 层级 | 技术选择 | 说明 |
|------|----------|------|
| 运行时 | **Tauri 2.x** | 一套代码，支持 Web + macOS + Windows + Linux |
| 后端 | Rust | 原生文件访问，高性能，体积小 |
| 前端框架 | React 18 + TypeScript | 组件化开发，类型安全 |
| 构建工具 | Vite | 快速开发体验 |
| 状态管理 | Zustand | 轻量级状态管理 |
| UI 样式 | Tailwind CSS + shadcn/ui | 快速构建美观界面 |
| Markdown | marked | Markdown 解析与渲染 |
| Frontmatter | gray-matter | YAML 元数据解析 |
| 搜索 | FlexSearch | 高性能全文搜索 |
| 编辑器 | CodeMirror 6 | 代码编辑体验 |
| 本地存储 | IndexedDB (Dexie.js) | Web 端缓存索引、配置数据 |
| 数据库 | SQLite (Rust 后端) | 桌面端索引存储 |

### 6.2 多端支持

| 平台 | 支持情况 | 说明 |
|------|----------|------|
| macOS | ✅ 原生支持 | 原生文件访问，系统通知 |
| Windows | ✅ 原生支持 | 原生文件访问，系统通知 |
| Linux | ✅ 原生支持 | 原生文件访问，系统通知 |
| Web (浏览器) | ✅ 支持 | Tauri 可构建为 Web 应用，使用 FSAA 访问文件 |

**核心优势**：前端代码 100% 复用，只需实现不同平台的文件访问适配层。

### 6.3 数据存储

- **Prompt 文件**：Markdown + YAML Frontmatter，与 Obsidian 格式兼容
- **存储位置**：用户本地目录（通过 `showDirectoryPicker()` 授权）
- **索引缓存**：IndexedDB 存储元数据和搜索索引
- **历史版本**：`.history/` 子目录（可选）

#### Prompt 文件格式示例

```markdown
---
title: "Linux 系统管理专家"
tags: [计算机, Linux, shell]
created: "2026-05-16T10:30:00Z"
modified: "2026-05-16T10:30:00Z"
copy_count: 3
pinned: false
---

你是一位 Linux 系统管理专家，精通命令行操作、系统配置和故障排查...
```

│  ┌─────────────────────────────────────────────────────┐    │
│  │              Services / UseCases                     │    │

**核心设计**：通过 IFileRepository 接口抽象文件访问，不同平台实现不同适配器，上层业务逻辑完全复用。

### 6.5 性能要求

| 指标 | 目标 |
|------|------|
| 首屏加载 | < 1 秒 |
| 搜索响应 | < 100ms |
| Prompt 切换 | < 300ms |
| 支持文件数 | 10,000+ |

### 6.6 安全与隐私

- 所有数据存储在用户本地，不经过服务器
- 文件访问需要用户明确授权
- 支持 PWA 安装到桌面
- 无追踪、无数据分析

### 6.7 技术风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| Tauri 生态较新 | 使用成熟稳定的前端技术栈，Rust 代码尽量简单 |
| Web 端兼容性 | Web 构建时检测 File System Access API 支持，不兼容时提示下载桌面版 |
| 文件并发写入 | 使用文件锁 + 写时复制 |
| 大量 Prompt 性能 | 虚拟滚动 + 分页加载 |
| 中文搜索效果 | FlexSearch 支持中文分词 |
| 多端同步（未来） | 预留云端同步接口 |

### 6.8 开发优先级

| 阶段 | 内容 |
|------|------|
| **Phase 1** | Tauri + React 基础框架，实现桌面端文件访问 |
| **Phase 2** | 核心功能：Prompt CRUD、标签管理、搜索 |
| **Phase 3** | Web 适配层，支持浏览器访问 |
| **Phase 4** | 高级功能：历史版本、批量操作、导入导出 |
| **Phase 5** | 云端同步（可选） |
