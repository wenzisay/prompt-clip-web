# 分享图功能同步至 iOS 版 — 实现指南

> 本文档供 agent 在 **iOS 代码库**中同步实现 Web 版于 2026-06-24 完成的两项分享图改动。
> 文档自包含：描述**行为契约 + 精确视觉规格 + 验收标准**，不依赖 Web 代码细节。
> 对应 Web 版 commit：`fd5f7d9`（选择性包含批注）、`3d1556a`（作者位置调整）。

---

## 0. 任务目标

iOS 版的「生成分享图片」功能需与 Web 版保持一致，共两项改动：

1. **选择性包含批注**：分享图可选择把 Prompt 的批注（文字 + 图片附件）一并放进卡片，可勾选包含哪几条。
2. **作者位置调整**：作者名称从卡片顶部移到底部右侧、紧邻 logo，两者之间加细竖线分隔符。

> 若 iOS 版当前**还没有**分享图功能或批注功能，先确认现状；本文假设两者已存在，仅需按下方契约调整。渲染方式（SwiftUI 截图 / Core Graphics 绘制 / UIKit）不限，但**视觉与交互契约必须一致**。

## 1. 改动一：选择性包含批注

### 1.1 交互行为

- 分享图生成界面（选项区）新增一个**总开关**：「包含批注 / Include annotations」。
- 默认**关闭**。
- **打开时默认全选**当前 Prompt 的所有批注；用户可逐条取消勾选。
- 当前 Prompt **无批注**时：开关**禁用**，并在下方显示提示「该 Prompt 暂无批注 / No annotations for this prompt」。
- 开关打开后，下方展开**批注勾选列表**：每条批注一个复选框 + 文字预览（截断 2 行）；若该批注带图片附件，显示图片数量提示（如 🖼️ 1）。
- 勾选变化**实时反映**在分享图预览中。

### 1.2 数据模型

```
批注（Annotation）：
  - id: String            // 稳定唯一标识
  - text: String          // 批注文字
  - attachments: [Attachment]   // 图片附件列表（可空）
附件（Attachment）：
  - id: String
  - name: String
  - mimeType: String      // 如 image/png
  - data: 二进制图片数据   // 本地存储，需读取后渲染

分享选项新增两个字段：
  - includeAnnotations: Bool          // 总开关
  - selectedAnnotationIds: [String]   // 选中的批注 id
```

**渲染规则**：按 `selectedAnnotationIds` 过滤批注，但**保持批注的原始顺序**（即批注列表的原顺序，**不**按勾选顺序）。

### 1.3 卡片渲染规格

**位置**：Prompt 正文（及「内容已截断」提示，若有）**之后**、Logo 区**之前**。仅当有选中的批注时才渲染整个区块。

**区块结构**（自上而下）：

```
┄┄┄┄┄┄ Note ┄┄┄┄┄┄        ← Note 分隔线（横向细线 + 中间小字 "Note"）

〔批注 1 文字〕
〔批注 1 图片（若有）〕

──────────                ← 批注之间的纯细线（无文字）

〔批注 2 文字〕
〔批注 2 图片（若有）〕
```

**关键约束**：
- **不要**「批注」之类的区块标题文字（用 Note 分隔线代替）。
- **不要**批注卡片的背景底纹 / 边框。批注文字直接以正文样式呈现。
- 批注区块宽度**与 Prompt 正文一致**（左右对齐正文，不额外缩进）。

### 1.4 精确视觉规格

> 数值来自 Web 版 CSS，可直接作为 pt 参照（Web 的 px ≈ iOS 的 pt）。颜色用「卡片当前文字色」的派生值，确保深色/浅色模板自适应。

| 元素 | 规格 |
|------|------|
| 批注区块与正文的间距 | 32pt |
| **Note 分隔线** | 左右两条 1pt 高横线，`当前文字色 @ 15% 不透明度`，中间夹「Note」字样 |
| **「Note」字样** | 10pt，medium 字重，字间距 +0.2em，`当前文字色 @ 40% 不透明度`，**保持英文 "Note" 不随界面语言变化**（装饰性） |
| Note 线与首批注间距 | 24pt |
| **批注文字** | 16pt，行高 28pt，颜色 = 卡片正文文字色，无背景 |
| **批注之间分隔线** | 1pt 横线，`当前文字色 @ 10% 不透明度`（比 Note 线更淡），与上下批注各距 20pt |
| **批注图片** | 宽度撑满正文区，圆角，与上方文字距 12pt |

> 「当前文字色」：极简白/淡彩模板为深灰文字 → 线呈浅灰；深色模板为浅色文字 → 线呈淡白。实现时取卡片正文的 `foregroundColor` 作为基色，按上述百分比叠加透明度即可。

### 1.5 图片附件处理

- 批注图片附件是**本地二进制**，分享图需**内联渲染**（读取数据 → UIImage）。
- 导出/截图**前必须确保所有选中批注的图片已加载完成**，否则导出的图片会缺图。

## 2. 改动二：作者位置调整

### 2.1 行为

- 作者名称**从卡片顶部移除**（顶部不再显示作者）。
- 作者名称改放到**卡片底部右侧**，位于 PromptClip logo 的**左边**，整行右对齐。
- 作者与 logo **之间**加一条**细竖线分隔符**，但**仅当两者同时显示时**才出现。

### 2.2 显示组合（必须全部正确）

| 显示作者 | 显示 Logo | 底部行内容 |
|:---:|:---:|---|
| ✅ | ✅ | `作者 │ [logo] PromptClip`（带竖线分隔符） |
| ✅ | ❌ | `作者` |
| ❌ | ✅ | `[logo] PromptClip` |
| ❌ | ❌ | 不渲染底部行 |

> 「显示作者」= 用户开启作者显示 **且** 作者名非空。

### 2.3 精确视觉规格

| 元素 | 规格 |
|------|------|
| 底部行与上方内容间距 | 40pt |
| 底部行对齐 | 右对齐，元素水平间距 12pt |
| 作者字号/字重 | 与「PromptClip」一致：约 14pt，medium 字重 |
| 整行不透明度 | 约 75% |
| **作者─Logo 分隔符** | 竖线，高 12pt，宽 1pt，`当前文字色 @ 40% 不透明度` |

## 3. iOS 实现提示（给 agent）

1. **定位代码**：在 iOS 仓库搜索分享图/分享卡片相关代码（关键词：share、card、export、image、render、snapshot）。同时定位批注相关代码（annotation）和分享选项 UI。
2. **渲染方式无关**：无论用 SwiftUI View 截图（`ImageRenderer`/`SnapshotKit` 等）还是 Core Graphics 手绘，**满足上方视觉契约即可**。若用 SwiftUI，分隔线可用 `Rectangle().fill(color.opacity(0.15)).frame(height: 1)`，「Note」用 `Text("Note").font(.system(size: 10))`。
3. **复用现有批注模型/加载**：iOS 若已有批注数据层，直接复用其模型与读取逻辑；只需新增「按 id 过滤 + 保持原序」的筛选。
4. **图片加载**：批注图片读取是异步的，导出前需等待（可用 `await` 或在图片就绪后再启用导出按钮）。
5. **多语言**：「包含批注」「该 Prompt 暂无批注」等界面文案需走 iOS 的本地化（`Localizable.strings` / String Catalog）；但卡片内的 **「Note」字样保持英文**，不本地化。
6. **三种模板**：若 iOS 有极简白 / 深色 / 淡彩边框三套卡片主题，逐个验证分隔线/文字颜色自适应正确。

## 4. 验收清单

### 批注功能
- [ ] 分享界面有「包含批注」总开关，默认关闭
- [ ] 无批注的 Prompt：开关禁用 + 显示「暂无批注」提示
- [ ] 打开开关：默认全选当前 Prompt 所有批注
- [ ] 可逐条勾选 / 取消，预览实时更新
- [ ] 导出的分享图包含**选中**的批注（文字 + 图片附件）
- [ ] 卡片内：有「Note」分隔线、批注之间有更淡的细线、**无**「批注」标题、**无**卡片底纹、批注宽度对齐正文
- [ ] 「Note」字样在各种界面语言下都是英文
- [ ] 三种卡片主题下视觉协调（线的浓淡随文字色自适应）

### 作者位置
- [ ] 作者不再出现在卡片顶部
- [ ] 作者在卡片底部右侧、logo 左侧
- [ ] 作者 + logo 同时显示时有竖线分隔符
- [ ] 仅作者 / 仅 logo / 两者皆无 三种组合都正确
- [ ] 三种卡片主题下视觉协调
